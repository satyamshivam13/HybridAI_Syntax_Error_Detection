**Status:** ✅ SHIPPED 2026-03-27

**Total Plans:** 10

> Historical snapshot note (added 2026-04-14): This archive captures v1.0-at-ship context. Some items listed as deferred technical debt were later addressed in v1.1 (see `.planning/ROADMAP.md`, `.planning/PROJECT.md`, and `.planning/UAT_AUDIT_REPORT_2026-04-14.md` for current status).

## Overview

OmniSyntax v1.0 hardened the hybrid detection pipeline for educational use. The milestone improved rule-based accuracy across C, Java, and JavaScript, established a metadata-aware ML compatibility contract, and delivered consistent degraded-mode behavior across the FastAPI, CLI, and Streamlit entry points. Students now receive accurate, actionable feedback regardless of whether the ML layer is available.

## Phases

### Phase 1: Detection Reliability
**Goal**: Make non-Python detection accurate enough to be trustworthy when the ML path is unavailable.
**Depends on**: Nothing (first phase)
**Plans**: 2 plans

Plans:
- [x] 01-01: Expand and normalize rule-based non-Python detection

## Milestone Summary

**Key Decisions:**
- Keep the brownfield Python architecture — focused improvements are lower risk than a rewrite ✓ Good
- Treat degraded-mode correctness as a first-class requirement — ML compatibility is fragile ✓ Good
- Organize work around detection reliability → feedback quality → ML recovery → QA hardening ✓ Good
- Use `.planning/codebase/` maps before deeper phase planning ✓ Good

**Issues Resolved:**
- Non-Python detection accuracy depending entirely on ML fallback
- False positives from bracket/delimiter checks inside string literals
- Inconsistent degraded-mode warnings across CLI, API, and Streamlit
- ML model bundle incompatibility with scikit-learn version

**Issues Deferred (v1.1+):**
- Parser-grade C/Java/JavaScript diagnostics (ADV-01)
- Richer semantic tutoring via alternative models (ADV-02)
- CLI indentation auto-fix doesn't rewrite code — only suggests (known gap)
- `--all-errors` mode misses ML-only errors like DivisionByZero for Python

**Technical Debt Incurred:**
- Auto-fix for IndentationError applies a comment suggestion rather than rewriting indentation
- UnclosedString auto-fix appends closing quote at end-of-file rather than on the original line
- False positive UndeclaredIdentifier sometimes accompanies TypeMismatch in Java
