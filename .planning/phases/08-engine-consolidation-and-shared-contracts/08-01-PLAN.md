<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| entry points -> shared engine | API, CLI, and Streamlit each cross into shared detection code with different presentation needs and must not diverge in result shape |
| compatibility layer -> authoritative pipeline | Legacy modules still wrap or forward to the engine and can reintroduce dead paths if parity is not locked |
| typed contract -> external JSON/UI surface | Internal typed results are transformed back into public outputs, so adapter bugs can silently distort behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|-----------|-----------|-------------|-----------------|
| T-08-01 | T | `src/error_engine.py` / `src/multi_error_detector.py` cleanup | mitigate | Remove dead branches only after parity tests pass and keep only minimal compatibility shims where required. |
| T-08-02 | I | shared orchestration helper and export surface | mitigate | Keep the helper co-located with `src/static_pipeline.py`, export it through `src/__init__.py`, and cover API/CLI/Streamlit parity tests. |
| T-08-03 | R | typed contract adapter layer | accept | Internal result transformation is bounded to this phase and reviewed by regression tests; no user identity or audit trail is in scope here. |
</threat_model>
