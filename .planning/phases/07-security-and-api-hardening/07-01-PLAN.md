<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external caller -> FastAPI boundary | Untrusted users submit arbitrary code and can abuse public routes if no explicit gate exists |
| process-local throttling -> deployed runtime topology | In-memory rate limiting drifts under multiple workers or instances |
| auto-fix output -> student/client trust | Generated fix suggestions can be mistaken for verified repairs if not re-analyzed |
| docs surface -> public internet | Interactive docs advertise the full API and should not be public by accident in production |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | S | `api.py` public routes | mitigate | Require explicit API-key-style access control in production-facing mode, with only a deliberate local/unsafe bypass. |
| T-07-02 | D | in-memory throttling | mitigate | Introduce a limiter abstraction with memory default and Redis-capable shared-store seam so the production path is not process-local fiction. |
| T-07-03 | T | `/fix` and `/check-and-fix` response contract | mitigate | Re-run shared detection on proposed fixes and return verification state instead of optimistic success flags. |
| T-07-04 | I | Swagger/ReDoc exposure | mitigate | Disable docs by default in production-facing configurations and document trusted-environment enablement explicitly. |
| T-07-05 | R | `/health` semantics | mitigate | Split liveness, readiness, and capabilities so operators can distinguish service availability from ML degradation. |
</threat_model>
